## 1. 类与对象

面向对象的核心思想是：**用"类"描述事物，用"对象"表示具体实例**。

> 前端类比：JS 的 `class` 语法和 Java 几乎一样，但 Java 更严格——所有代码必须写在类里。

### 1.1 定义类与创建对象

```java
// 定义类
public class Person {
    // 属性（成员变量）
    String name;
    int age;
    long id;
    double salary;
    boolean isStudent;
    char gender;

    // 方法
    public void sayHello() {
        System.out.println("你好，我叫 " + name + "，今年 " + age + " 岁。");
    }
}
```

```java
// 创建对象（实例化）
public class Main {
    public static void main(String[] args) {
        Person p = new Person();   // new 创建对象
        p.name = "张三";
        p.age = 25;
        p.sayHello();              // 你好，我叫 张三，今年 25 岁。
    }
}
```

### 1.2 构造方法（Constructor）

构造方法是**创建对象时自动调用**的特殊方法，方法名与类名相同，没有返回值。

```java
public class Person {
    String name;
    int age;

    // 无参构造方法
    public Person() {
        this.name = "未知";
        this.age = 0;
    }

    // 有参构造方法
    public Person(String name, int age) {
        this.name = name;   // this 指当前对象，类似 JS 中的 this
        this.age = age;
    }

    public void sayHello() {
        System.out.println("你好，我叫 " + name + "，今年 " + age + " 岁。");
    }
}
```

```java
// 使用不同构造方法
Person p1 = new Person();                  // 调用无参构造
Person p2 = new Person("李四", 30);        // 调用有参构造

p1.sayHello();   // 你好，我叫 未知，今年 0 岁。
p2.sayHello();   // 你好，我叫 李四，今年 30 岁。
```

> ⚠️ 如果你没有定义任何构造方法，Java 会自动提供一个无参构造方法。一旦你定义了有参构造方法，无参构造方法就**不会自动提供**，需要手动添加。

### 1.3 this 关键字

`this` 指向当前对象本身，常用于：

1. 区分成员变量和参数同名的情况
2. 在一个构造方法中调用另一个构造方法

```java
public class Person {
    String name;
    int age;
    String city;

    // 两参数构造方法
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
        this.city = "北京";   // 默认城市
    }

    // 三参数构造方法，调用上面的两参数构造
    public Person(String name, int age, String city) {
        this(name, age);    // 调用同类的另一个构造方法，必须写在第一行
        this.city = city;
    }
}
```

## 2. 封装

封装是 OOP 的第一大特性：**把数据（属性）隐藏在类内部，只通过方法对外暴露**。

> 好处：保护数据不被随意修改，比如 age 不能被设置为负数。

### 2.1 访问修饰符

| 修饰符      | 当前类 | 同包 | 子类 | 其他类 |
| ----------- | ------ | ---- | ---- | ------ |
| `public`    | ✅     | ✅   | ✅   | ✅     |
| `protected` | ✅     | ✅   | ✅   | ❌     |
| （无修饰）  | ✅     | ✅   | ❌   | ❌     |
| `private`   | ✅     | ❌   | ❌   | ❌     |

### 2.2 private + getter/setter

```java
public class Person {
    private String name;   // private：外部不能直接访问
    private int age;

    // getter：获取属性值
    public String getName() {
        return name;
    }

    // setter：设置属性值，可加入校验逻辑
    public void setName(String name) {
        if (name == null || name.isEmpty()) {
            System.out.println("姓名不能为空！");
            return;
        }
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age < 0 || age > 150) {
            System.out.println("年龄不合法！");
            return;
        }
        this.age = age;
    }
}
```

```java
Person p = new Person();
p.setName("王五");
p.setAge(25);
p.setAge(-5);             // 年龄不合法！

System.out.println(p.getName());   // 王五
System.out.println(p.getAge());    // 25
```

> 💡 IntelliJ IDEA 快速生成 getter/setter：右键 → Generate → Getter and Setter

## 3. 继承

继承是 OOP 的第二大特性：**子类可以复用父类的属性和方法，并扩展自己的功能**。

### 3.1 extends

```java
// 父类
public class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " 在吃东西");
    }

    public void sleep() {
        System.out.println(name + " 在睡觉");
    }
}
```

```java
// 子类：继承 Animal
public class Dog extends Animal {
    String breed;   // 狗特有的属性：品种

    public Dog(String name, String breed) {
        super(name);         // 调用父类构造方法，必须写在第一行
        this.breed = breed;
    }

    // 子类特有的方法
    public void bark() {
        System.out.println(name + " 汪汪叫！");
    }
}
```

```java
Dog dog = new Dog("旺财", "拉布拉多");
dog.eat();    // 旺财 在吃东西（继承自父类）
dog.sleep();  // 旺财 在睡觉（继承自父类）
dog.bark();   // 旺财 汪汪叫！（子类特有）
```

### 3.2 方法重写（Override）

子类可以**重写**父类的方法，提供自己的实现。

```java
public class Cat extends Animal {

    public Cat(String name) {
        super(name);
    }

    // @Override 注解：声明这是重写父类方法（编译器会检查）
    @Override
    public void eat() {
        System.out.println(name + " 优雅地吃鱼");
    }

    public void meow() {
        System.out.println(name + " 喵喵叫~");
    }
}
```

```java
Cat cat = new Cat("咪咪");
cat.eat();    // 咪咪 优雅地吃鱼（子类重写后的版本）
cat.sleep();  // 咪咪 在睡觉（继承自父类）
cat.meow();   // 咪咪 喵喵叫~
```

### 3.3 super 关键字

`super` 指向父类，有两种用法：

```java
public class Dog extends Animal {

    public Dog(String name) {
        super(name);    // 1. 调用父类构造方法
    }

    @Override
    public void eat() {
        super.eat();    // 2. 调用父类方法
        System.out.println(name + " 还想再来一碗！");
    }
}
```

### 3.4 Java 继承的限制

- Java **只支持单继承**：一个类只能有一个直接父类（`extends` 只能写一个）
- 所有类都隐式继承自 `Object` 类（Java 中一切皆对象）

## 4. 多态

多态是 OOP 的第三大特性：**同一个方法，不同对象调用时表现不同**。

### 4.1 向上转型

子类对象可以赋值给父类类型的变量（自动转换，安全）。

```java
Animal a1 = new Dog("旺财", "金毛");   // Dog 是 Animal，向上转型
Animal a2 = new Cat("咪咪");

a1.eat();   // 旺财 在吃东西（Dog 没重写 eat，调用父类）
a2.eat();   // 咪咪 优雅地吃鱼（Cat 重写了 eat，调用子类版本）
```

> 核心规则：调用哪个方法，取决于**对象的实际类型**，而不是变量的声明类型。这就是多态！

### 4.2 多态的实际用途

```java
public class AnimalShelter {

    // 参数类型是父类 Animal，可以接收任意子类对象
    public static void feedAnimal(Animal animal) {
        animal.eat();
    }

    public static void main(String[] args) {
        feedAnimal(new Dog("旺财", "柴犬"));   // 旺财 在吃东西
        feedAnimal(new Cat("咪咪"));            // 咪咪 优雅地吃鱼
    }
}
```

### 4.3 向下转型与 instanceof

向上转型后，如果要调用子类特有的方法，需要**向下转型**（强制转换）。

```java
Animal a = new Dog("旺财", "哈士奇");

// 向下转型前，先用 instanceof 判断，防止 ClassCastException
if (a instanceof Dog) {
    Dog d = (Dog) a;    // 向下转型
    d.bark();           // 旺财 汪汪叫！
}

// Java 16+ 新写法（模式匹配，推荐）
if (a instanceof Dog d) {
    d.bark();   // 直接使用，更简洁
}
```

### 4.4 方法重载（Overload）

重载是同一个类中**同名方法、不同参数**的情况（与重写不同，不涉及继承）。

```java
public class MathUtils {

    public static int add(int a, int b) {
        return a + b;
    }

    public static double add(double a, double b) {
        return a + b;
    }

    public static int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

```java
System.out.println(MathUtils.add(1, 2));          // 3
System.out.println(MathUtils.add(1.5, 2.5));      // 4.0
System.out.println(MathUtils.add(1, 2, 3));        // 6
```

> 重写（Override）vs 重载（Overload）：
>
> - **重写**：子类修改父类方法的实现，方法签名完全相同
> - **重载**：同一类中同名方法，参数不同（类型/数量）

## 5. 抽象类

抽象类是**不能被实例化的类**，用来定义规范，强制子类必须实现某些方法。

```java
// abstract 修饰的类是抽象类
public abstract class Shape {
    String color;

    public Shape(String color) {
        this.color = color;
    }

    // 抽象方法：只有声明，没有方法体，子类必须实现
    public abstract double area();
    public abstract double perimeter();

    // 普通方法：子类直接继承
    public void describe() {
        System.out.println("这是一个 " + color + " 色的图形");
        System.out.printf("面积：%.2f，周长：%.2f%n", area(), perimeter());
    }
}
```

```java
// 子类必须实现所有抽象方法
public class Circle extends Shape {
    double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }

    @Override
    public double perimeter() {
        return 2 * Math.PI * radius;
    }
}
```

```java
public class Rectangle extends Shape {
    double width, height;

    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }

    @Override
    public double area() {
        return width * height;
    }

    @Override
    public double perimeter() {
        return 2 * (width + height);
    }
}
```

```java
Shape c = new Circle("红", 5);
Shape r = new Rectangle("蓝", 4, 6);

c.describe();   // 这是一个 红 色的图形  面积：78.54，周长：31.42
r.describe();   // 这是一个 蓝 色的图形  面积：24.00，周长：20.00

// Shape s = new Shape("绿");  // ❌ 报错！抽象类不能实例化
```

## 6. 接口

接口是**比抽象类更纯粹的规范**，只定义"能做什么"，不关心"怎么做"。

### 6.1 定义与实现接口

```java
// 用 interface 定义接口
public interface Flyable {
    // 接口中的方法默认是 public abstract（可以省略不写）
    void fly();
    void land();
}

public interface Swimmable {
    void swim();
}
```

```java
// 用 implements 实现接口，一个类可以实现多个接口（弥补单继承的限制）
public class Duck extends Animal implements Flyable, Swimmable {

    public Duck(String name) {
        super(name);
    }

    @Override
    public void fly() {
        System.out.println(name + " 扑扇翅膀飞起来了");
    }

    @Override
    public void land() {
        System.out.println(name + " 落地了");
    }

    @Override
    public void swim() {
        System.out.println(name + " 在水里游泳");
    }
}
```

```java
Duck duck = new Duck("唐老鸭");
duck.eat();    // 继承自 Animal
duck.fly();    // 实现 Flyable 接口
duck.swim();   // 实现 Swimmable 接口
```

### 6.2 接口的默认方法（Java 8+）

Java 8 开始，接口可以有**默认方法**（有具体实现，不强制重写）。

```java
public interface Flyable {
    void fly();

    // 默认方法：子类可以直接使用，也可以重写
    default void glide() {
        System.out.println("滑翔中...");
    }
}
```

### 6.3 接口 vs 抽象类

| 对比项    | 抽象类               | 接口                                |
| --------- | -------------------- | ----------------------------------- |
| 关键字    | `abstract class`     | `interface`                         |
| 实现/继承 | `extends`（单继承）  | `implements`（可多实现）            |
| 成员变量  | 可以有普通变量       | 只能有常量（`public static final`） |
| 构造方法  | 有                   | 无                                  |
| 普通方法  | 可以有具体实现       | Java 8+ 支持 `default` 方法         |
| 使用场景  | 有公共代码需要复用时 | 定义能力/行为规范时                 |

> 经验法则：**is-a 关系用继承**（Dog is an Animal），**can-do 关系用接口**（Duck can fly）

## 7. 静态成员

`static` 修饰的成员属于**类本身**，而不是某个对象。所有对象共享同一份静态数据。

### 7.1 静态变量

```java
public class Counter {
    // 静态变量：所有 Counter 对象共享
    private static int count = 0;
    private int id;

    public Counter() {
        count++;          // 每创建一个对象，计数加一
        this.id = count;
    }

    public static int getCount() {
        return count;
    }

    public int getId() {
        return id;
    }
}
```

```java
Counter c1 = new Counter();
Counter c2 = new Counter();
Counter c3 = new Counter();

System.out.println(Counter.getCount());   // 3（通过类名访问静态方法）
System.out.println(c1.getId());           // 1
System.out.println(c2.getId());           // 2
System.out.println(c3.getId());           // 3
```

### 7.2 静态方法

静态方法**不依赖对象**，直接通过类名调用，无法访问非静态成员。

```java
public class StringUtils {

    // 工具类方法通常都是 static 的
    public static String reverse(String s) {
        return new StringBuilder(s).reverse().toString();
    }

    public static boolean isPalindrome(String s) {
        String reversed = reverse(s);
        return s.equalsIgnoreCase(reversed);
    }
}
```

```java
System.out.println(StringUtils.reverse("Hello"));       // olleH
System.out.println(StringUtils.isPalindrome("madam"));  // true
System.out.println(StringUtils.isPalindrome("hello"));  // false
```

### 7.3 静态代码块

静态代码块在**类加载时执行一次**，常用于初始化静态变量。

```java
public class Config {
    public static final String APP_NAME;
    public static final int MAX_CONNECTIONS;

    static {
        // 类加载时执行，只执行一次
        APP_NAME = "MyApp";
        MAX_CONNECTIONS = 100;
        System.out.println("Config 类已加载");
    }
}
```

## 8. 常用类

### 8.1 String 与 StringBuilder

```java
// String 是不可变的，每次修改都会创建新对象
String s = "Hello";
s = s + " World";   // 实际上创建了新的 String 对象

// StringBuilder 是可变的，适合频繁拼接字符串
StringBuilder sb = new StringBuilder();
sb.append("Hello");
sb.append(" ");
sb.append("World");
sb.insert(5, ",");         // 在索引 5 处插入
sb.delete(0, 5);           // 删除 [0, 5) 范围内的字符
sb.reverse();              // 反转
System.out.println(sb.toString());   // dlroW

// 常用方法对比
String str = "  Hello, Java!  ";
System.out.println(str.trim());             // "Hello, Java!"，去首尾空格
System.out.println(str.strip());            // Java 11+，更好的 trim（支持 Unicode 空白）
System.out.println(str.isBlank());          // false
System.out.println("  ".isBlank());         // true，Java 11+
System.out.println(str.repeat(2));          // Java 11+，重复字符串
System.out.println(String.join("-", "a", "b", "c"));  // "a-b-c"
```

### 8.2 Math 类

```java
System.out.println(Math.abs(-10));       // 10，绝对值
System.out.println(Math.max(3, 7));      // 7，最大值
System.out.println(Math.min(3, 7));      // 3，最小值
System.out.println(Math.pow(2, 10));     // 1024.0，幂运算
System.out.println(Math.sqrt(16));       // 4.0，平方根
System.out.println(Math.floor(3.9));     // 3.0，向下取整
System.out.println(Math.ceil(3.1));      // 4.0，向上取整
System.out.println(Math.round(3.5));     // 4，四舍五入

// 随机数
double r = Math.random();               // [0.0, 1.0) 之间的随机数
int dice = (int)(Math.random() * 6) + 1; // 模拟骰子：1~6
```

### 8.3 包装类（Wrapper Class）

Java 的基本类型不是对象，但集合等框架要求必须用对象。包装类将基本类型"包装"成对象。

| 基本类型  | 包装类      |
| --------- | ----------- |
| `int`     | `Integer`   |
| `long`    | `Long`      |
| `double`  | `Double`    |
| `float`   | `Float`     |
| `boolean` | `Boolean`   |
| `char`    | `Character` |
| `byte`    | `Byte`      |
| `short`   | `Short`     |

```java
// 装箱：基本类型 → 包装类（Java 5+ 自动装箱）
Integer a = 10;           // 自动装箱，等价于 Integer.valueOf(10)
int b = a;                // 自动拆箱，等价于 a.intValue()

// 常用方法
System.out.println(Integer.parseInt("123"));     // 123，字符串转 int
System.out.println(Integer.toBinaryString(10));  // "1010"，十进制转二进制
System.out.println(Integer.toHexString(255));    // "ff"，十进制转十六进制
System.out.println(Integer.MAX_VALUE);           // 2147483647
System.out.println(Integer.MIN_VALUE);           // -2147483648

System.out.println(Double.parseDouble("3.14"));  // 3.14
System.out.println(Boolean.parseBoolean("true")); // true
```

### 8.4 LocalDate / LocalDateTime（现代日期 API，Java 8+）

> ⚠️ 不推荐再用旧的 `Date` 和 `Calendar`，Java 8 引入了更好用的 `java.time` 包。

```java
import java.time.LocalDate;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

// 日期
LocalDate today = LocalDate.now();
System.out.println(today);                        // 2026-04-28
System.out.println(today.getYear());              // 2026
System.out.println(today.getMonthValue());        // 4
System.out.println(today.getDayOfMonth());        // 28

LocalDate birthday = LocalDate.of(2000, 1, 1);
System.out.println(today.isAfter(birthday));      // true

// 日期时间
LocalDateTime now = LocalDateTime.now();
System.out.println(now);                          // 2026-04-28T10:30:00.000

// 格式化
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("yyyy年MM月dd日 HH:mm:ss");
System.out.println(now.format(fmt));              // 2026年04月28日 10:30:00

// 解析
LocalDate date = LocalDate.parse("2026-04-28");
LocalDateTime dt = LocalDateTime.parse("2026-04-28T10:30:00");
```

for (Animal a : animals) {

// 要求实现以下图形：Circle、Rectangle、Triangle

### 练习三：学生管理类

设计一个 `Student` 类：

## 知识点速查

| 概念         | 关键字 / 用法                   | 说明                       |
| ------------ | ------------------------------- | -------------------------- |
| 定义类       | `class Person {}`               | 所有代码必须在类里         |
| 创建对象     | `Person p = new Person()`       | `new` 在堆内存分配空间     |
| 当前对象     | `this`                          | 类似 JS 的 `this`          |
| 父类引用     | `super`                         | 调用父类构造/方法          |
| 继承         | `class Dog extends Animal`      | 单继承                     |
| 实现接口     | `class Duck implements Flyable` | 可多实现                   |
| 方法重写     | `@Override`                     | 子类覆盖父类方法           |
| 抽象类       | `abstract class Shape`          | 不能实例化，可以有抽象方法 |
| 抽象方法     | `abstract void area()`          | 只有声明，子类必须实现     |
| 接口         | `interface Flyable`             | 定义行为规范               |
| 静态成员     | `static`                        | 属于类，不属于对象         |
| 类型判断     | `a instanceof Dog`              | 判断对象是否是某类型       |
| 私有属性     | `private String name`           | 外部不可直接访问           |
| 访问私有属性 | `getName()` / `setName()`       | 通过 getter/setter         |

## OOP 四大特性总结

```
封装  →  保护数据，控制访问（private + getter/setter）
继承  →  代码复用，建立类层级（extends）
多态  →  统一接口，不同实现（向上转型 + 方法重写）
抽象  →  定义规范，强制实现（abstract class / interface）
```
